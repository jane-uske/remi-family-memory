# v1.3 Checkpoint: Source / Evidence / Confidence Model

**Released:** 2026-05-19
**Scope:** Minimal provenance tracking on confirmed memories — source type, OCR/VLM assistance flags, parent confirmation timestamp

---

## What v1.3 adds

Every confirmed memory now carries a `provenance` field tracing how it was created:

| Field | Type | Description |
|-------|------|-------------|
| `sourceType` | `'remi_capture' \| 'asset_intake' \| 'folder_scan' \| 'manual'` | How the data entered the system |
| `confidence` | `'confirmed_by_parent'` | Only value — parent confirmation is the sole confidence source |
| `draftId` | `string?` | Original draft ID if from asset intake |
| `originalFilenames` | `string[]?` | Source files if applicable |
| `ocrAssisted` | `boolean` | Whether OCR was used during processing |
| `vlmAssisted` | `boolean` | Whether VLM was used during processing |
| `vlmModel` | `string?` | Which VLM model was used (if any) |
| `confirmedAt` | `string` | ISO timestamp of parent confirmation |

---

## Data model

### EventProvenance (on BabyEvent)

Written into inbox note frontmatter at `confirmDraft()` time:

```yaml
provenance:
  sourceType: asset_intake
  draftId: "draft-abc-123"
  ocrUsed: true
  vlmUsed: true
  vlmModel: "qwen-vl-7b"
  confirmedAt: "2026-05-19T10:00:00.000Z"
```

Parsed by `parseMarkdownNote()` into `BabyEvent.provenance`.

### MemoryProvenance (on MemoryRecord)

Derived from EventProvenance by `eventToMemory()`:

```json
{
  "sourceType": "asset_intake",
  "confidence": "confirmed_by_parent",
  "draftId": "draft-abc-123",
  "ocrAssisted": true,
  "vlmAssisted": true,
  "vlmModel": "qwen-vl-7b",
  "confirmedAt": "2026-05-19T10:00:00.000Z"
}
```

## Provenance in Remi answers

When Remi answers a family memory question, the response now includes a brief provenance note:

| Scenario | Note appended to answer |
|----------|------------------------|
| Plain confirmed memory (no OCR/VLM) | `（已由家长确认）` |
| OCR-assisted memory | `（已由家长确认，OCR 辅助提取）` |
| VLM-assisted memory | `（已由家长确认，VLM 辅助整理）` |
| Both OCR + VLM | `（已由家长确认，OCR + VLM 辅助整理）` |
| No provenance on memory | No note appended |

Key language choices:
- VLM is described as "辅助整理" (assisted organizing), never as "证明" or "确认"
- OCR is described as "辅助提取" (assisted extraction)
- Parent confirmation is always the primary attribution

## What provenance does NOT do

| Action | Allowed? | Reason |
|--------|----------|--------|
| Change confirmed-only guard | **No** | `confirmedByParent === true` is still the only gate |
| Allow unconfirmed memory to be queried | **No** | provenance is only written on confirmed path |
| Provide numerical confidence scores | **No** | Only `confirmed_by_parent` exists |
| Auto-confirm based on VLM confidence | **No** | VLM output is advisory; human confirms |
| Show provenance to LLM as a trust signal | **No** | provenance is for the human, not the AI |

## Pipeline flow

```
Image uploaded → pending draft
       ↓
OCR extracts text (ocrUsed=true if successful)
       ↓
VLM enriches draft (vlmUsed=true if ran)
       ↓
Parent confirms → confirmDraft() writes provenance into inbox note YAML
       ↓
npm run sync → parseMarkdownNote() → BabyEvent with provenance
       ↓
buildMemories() → MemoryRecord with provenance
       ↓
Remi queries → connector looks up provenance → appends note to answer
```

## Backward compatibility

- Events/memories without `provenance` field continue to work — the field is optional
- Old confirmed notes (pre-v1.3) produce `provenance: undefined` — no note is appended
- No migration needed for existing data

## Boundaries for v1.3

Done:
- EventProvenance and MemoryProvenance types
- confirmDraft writes provenance to inbox note
- parseMarkdownNote reads provenance from frontmatter
- eventToMemory carries provenance to MemoryRecord
- Connector looks up provenance when answering
- formatProvenanceNote generates brief Chinese annotation
- Remi capability appends provenance note to answer

Not done (future):
- Numerical confidence scoring
- Multi-source provenance (combining manual + VLM evidence)
- Provenance displayed in web dashboard
- "来源" intent detection in Remi (e.g., "这条记录怎么来的")
- Provenance-based answer ranking
