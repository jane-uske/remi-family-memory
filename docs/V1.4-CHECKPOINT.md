# v1.4 Checkpoint: Family Memory Review Experience

**Released:** 2026-05-19
**Scope:** Minimal owner-facing review page for confirmed memories and pending drafts

---

## What v1.4 adds

A single-page review experience at `/review.html` that lets the parent:

1. **View confirmed memories** — sorted by date, with provenance annotations
2. **View pending drafts** — with VLM enrichment status and file list
3. **Confirm or reject drafts** — directly from the web UI
4. **Add summary** — optional text input before confirming
5. **See provenance** — each confirmed memory shows source attribution

---

## UI: Review Page

Accessible at `http://localhost:3456/review.html`

### Pending tab (default)
- Orange-bordered cards for each pending draft
- Shows inferred date, type, title, file list
- VLM enrichment indicated with green checkmark and fact preview
- Textarea for optional summary
- "确认" and "跳过" buttons

### Confirmed tab
- White cards sorted newest-first
- Shows date, type, importance badge, title, summary, top facts
- Provenance note at bottom: `✓ 家长已确认 · VLM 辅助整理 · 2026-05-19 确认`

---

## New Owner-Facing API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/memories` | GET | All confirmed memories (with provenance) |
| `/api/drafts/pending` | GET | Pending drafts only |
| `/api/drafts/:id/confirm` | POST | Confirm a draft (owner, no auth) |
| `/api/drafts/:id/reject` | POST | Reject a draft (owner, no auth) |

These mirror the AI-facing endpoints but without token requirement — intended for the local web dashboard.

---

## What v1.4 does NOT do

| Feature | Status | Reason |
|---------|--------|--------|
| Auth on owner endpoints | Not added | Local-first: owner dashboard runs on private network |
| Edit confirmed memories | Not added | Confirmed is immutable; re-import to correct |
| Batch confirm all | Not added | Each draft needs individual review |
| Auto-refresh | Not added | Manual page reload |
| Mobile-responsive polish | Minimal | Basic viewport meta, no dedicated mobile design |
| File preview / thumbnails | Not added | Future: show attached images inline |
| Timeline filtering | Not added | Future: filter by type, date range |
| Gestational week labels | Not added | Available via /api/profile, can be added |

---

## Safety invariants preserved

1. **confirmed-only guard**: pending drafts stay pending until explicit parent confirm via button click
2. **No auto-confirm**: web UI requires user action per draft
3. **No AI bypass**: owner-facing endpoints don't feed into Remi query path
4. **VLM is "辅助整理"**: never presented as authoritative
